The one rule that decides everything

Blend mode for bright. True alpha for solid.

This is how you brief whoever makes the assets. Get it wrong and people turn see-through.

SCREEN BLEND

Bright effects on black

Fire · flames · sparks · light · water spray. Shot on black, screened on. Works everywhere incl. iPad. Lightweight.

fire.mp4 → blend: screen → black drops out
TRUE ALPHA

Solid objects

People · vehicles · triage area · props · dark smoke. These would go faint with screen blend — they need real transparency (packed-alpha MP4).

officer.mp4 → packed-alpha → cut out cleanly

How a casualty plays out.

Drag-and-drop assets that fade in on cue. The instructor voices it over Teams; the visuals just need to land.

1

Casualty appears

Fades in by the building. Coughing loop + cough sound plays.

2

"Set up triage"

Triage-area asset (green sheet + kit) fades in where pointed.

3

Move casualty

Walk-clip plays → casualty now stood at triage. Triaged.

4

"Speak to ambulance"

Paramedic walk-in plays → stands close. Conversation (live voice).

5

Dismiss

"Carry on" → paramedic walks off, fades out.

Your job, as the builder

How you add an asset.

The admin flow you'll live in — upload once, set its looks, file it. This is the back-end that feeds everything instructors do.

STEP 1

Upload

Drop the clip/PNG. Pick source: bright-on-black, white, or true-alpha.

STEP 2

Categorise

Category + subcategory (People ▸ Police).

STEP 3

Add looks

Attach variants + states (idle, walk-in, kneeling).

STEP 4

Set playback

Default once / loop / hold + blend mode.

STEP 5

Save

Lands in the Library, ready to drag.

+

Templates work the same way. Save a whole built scene (house + fire + crew) as a template; reload it, swap the house image, and the scenario still runs — different property, same hard work reused.
