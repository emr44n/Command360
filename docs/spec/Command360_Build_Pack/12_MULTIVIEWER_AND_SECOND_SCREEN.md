# 12 · Multiviewer & Second Screen

## 1. Multiviewer (broadcast vision-mixer)

A vision-mixer for the instructor to cut between camera **Views** live.

- **View tiles** in a grid.
- **Hotkeys 1–9** / click to **cut** the active View to the candidate **Stage** (what participants see).
- **Split / grid** output options.
- The instructor manipulates the underlying View live → it reflects on the participant Stage.

## 2. CCTV as a slide type

- **CCTV is a View/slide type with a selectable source.**
- The instructor manipulates the underlying slide live → it shows on the "CCTV screen".
- Each event is a **layer** on top (build order = z-order).

### Label set (researched)
A label sits on a coloured bar over the feed:
`CCTV CAM · BODYCAM · DASHCAM · DRONE · AIR SUPPORT · THERMAL · NIGHT VISION · LIVE · ON AIR · BREAKING` + **custom** (text on a coloured bar).

## 3. Second screen (participant phones)

- Participants join via **link / QR + numeric code**.
- Their **phone is a second screen** — receives Cards (file 09), and can be the device that submits a Decision (file 10).
- No accounts; Group name only (file 05).
- Realtime sync via the existing event loop (file 14).

## 4. Title / loading screen

Before a session starts, participants see a **title/loading screen** (org logo, session title, "loading…") — not the live content. Pressing **Present** starts the session and opens joining.
