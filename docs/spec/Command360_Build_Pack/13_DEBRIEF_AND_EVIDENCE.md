# 13 · Debrief & Evidence

Debrief is the third mode (`Build → Run → Debrief`) and its own IA section (reports + analytics merged here — file 03).

---

## 1. Debrief mode

After a session, the instructor reviews:
- **Decision logs** — what each Group/participant decided, when (from Decision blocks / the live run).
- **Timeline replay** — the sequence of injects and decisions over time.
- **Analytics** — simple per-session views (participation, choices made).
- **AI debrief summaries** — Gemini-generated summary of the session (no personal data sent).

## 2. Decision logs — keep or wipe

- **Default: auto-wipe** at session end (the privacy selling point — participants hold no accounts/PII).
- Configurable per org to **keep + export** (for evidence).

## 3. Evidence export (the deal-closer)

> Your backlog calls this **"the single most important commercial feature."** It's what closes COMAH and aviation, because regulated buyers must *evidence* the exercise (COMAH 3-yearly tested exercise, CAP 699, Fire Standards, NHS EPRR).

**Spec:**
- Auto-generate an **exercise record**: timeline of injects & decisions, participant/Group performance vs objectives, attendance, instructor notes.
- Export to **PDF and spreadsheet (CSV/XLSX)**.
- **Brandable** with the org's logo (white-label).
- Fits the positioning: it **records and exports** the exercise — it does **not** auto-mark competency.

## 4. Optional objectives/scoring layer (later)

- **Objectives** (assessment criteria) can be attached to a Scenario/TDE and mapped to a framework (THINCS, CAP 699, COMAH, EPRR).
- An **assessor scoring view** can feed results into the evidence export.
- This is **optional and later** — Command 360's core is delivery + the evidence record, not marking.

## 5. Export hooks (later)

- Optional **LMS / competency-system export hooks** so outcomes can feed an external system of record (mirrors how incumbents cross-sell competency platforms).
